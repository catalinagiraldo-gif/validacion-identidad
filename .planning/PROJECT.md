# RPP Hub — Dropi Prototyping Hub

## What This Is

A private web portal at `dropitesters.co` where stakeholders (sponsors, board members, Product Designers, devs, external testers) can browse and interact with live Angular prototypes built by the RPP pipeline. Users log in with Google, select a profile (Dropshipper, Proveedor, or Admin), and see only the prototypes relevant to that role.

## Core Value

Stakeholders enter the Hub, pick their profile, and access any live prototype in one click — no setup, no git, no local environment needed.

## Requirements

### Validated

- Hub home page with grid of prototype cards, sorted by date added descending
- Badge "Nuevo" on recently added prototypes
- Search bar for finding prototypes
- Login via Google SSO with allowlist (auto-approve `@dropi.co`, manual allowlist for external emails)
- Profile selection screen post-login: Dropshipper, Proveedor, Admin (3 cards)
- Hub filtered by selected profile — only shows prototypes tagged for that role
- Profile selection per session (not persisted — choose every visit)
- Switch profile button in header
- Layout: Dropi chrome (sidebar 250px + header 70px)
- Carpeta-por-prototipo structure with `meta.json` per prototype
- Registry autogenerated in CI from `meta.json` files (not committed)
- Routes autogenerated in CI from `meta.json` files
- Thumbnails auto-captured by Playwright in CI (manual `thumbnail.png` override respected)
- CI validation: `meta.json` schema, route collision check, Angular build pass
- Auto-PR created by GitHub Actions on push to `prototype/DROP-XXXX-*` branches (with dedup — skips if PR already exists)
- `yarn nuevo-prototipo` interactive scaffolding generator
- Deploy to Vercel at `dropitesters.co`
- Claude Code handles all git operations for designers ("Subir PR a git" command)

### Active

(All above are active — building toward these)

### Out of Scope

- CTA "Dar feedback" on prototype cards — posponed, not cancelled
- Filtros por módulo/tags in Hub — only search bar + sort by date for v1
- Real backend / database — everything is client-side with mock data and JSON files
- Custom auth system — using Google Identity Services, no email/password
- Mobile app — web only
- Role-based access control (all authenticated users see all profiles) — anyone who passes the allowlist can pick any profile
