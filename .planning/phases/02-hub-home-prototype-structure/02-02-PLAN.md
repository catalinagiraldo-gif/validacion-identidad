Create `scripts/generate-registry.ts` as a Node.js script (per D-20). This script uses ONLY Node.js built-in modules (fs, path, glob via fs.readdirSync recursion) -- NO external dependencies. It will be executed via `npx ts-node scripts/generate-registry.ts` or as plain `.js` if ts-node is not available.

**Script logic:**

1. Scan `src/app/pages/` recursively for all files named `meta.json`
2. Read and parse each meta.json
3. Validate that each has the 9 required fields: title, module, slug, route, description, profiles, owner, dateAdded, thumbnail
4. If validation fails, print error with file path and missing fields, exit with code 1
5. Sort entries alphabetically by `module` then by `title`
6. Write output to `src/app/config/prototypes.registry.ts` with this format:

```typescript
// AUTO-GENERATED by scripts/generate-registry.ts
// Do not edit manually. Run: yarn generate-registry
import { PrototypeMeta } from '../models/prototype-meta.model';

export const PROTOTYPE_REGISTRY: PrototypeMeta[] = [
  {
    title: 'Academy',
    module: 'standalone',
    slug: 'academy',
    route: '/academy',
    description: 'Centro de formacion...',
    profiles: ['dropshipper'],
    owner: 'producto@dropi.co',
    dateAdded: '2026-05-01',
    thumbnail: 'thumbnail.png',
  },
  // ... remaining entries
];
```

7. Print summary: "Generated prototypes.registry.ts with N prototypes"

**IMPORTANT implementation detail:** Write the script as a `.ts` file but use only CommonJS-compatible syntax (`require`, `module.exports` where needed) since the project does not have ts-node configured. The actual execution will be: `node --require ts-node/register scripts/generate-registry.ts` OR the fallback approach is to write it as plain JS in a `.ts` file extension (just use `const fs = require('fs')` etc). Choose the simplest approach that works: write it as `scripts/generate-registry.js` (plain Node.js) and name the file `.js`. Then add a yarn script.

Actually, per Claude's Discretion on script language: **use plain .js** for maximum compatibility. Create `scripts/generate-registry.js` instead. No ts-node dependency needed.
