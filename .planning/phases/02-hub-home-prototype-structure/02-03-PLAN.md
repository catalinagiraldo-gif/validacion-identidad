**Component logic (TypeScript):**

```typescript
// Imports needed:
import { Component, OnInit, OnDestroy } from '@angular/core';
import { CommonModule } from '@angular/common';
import { RouterModule } from '@angular/router';
import { FormsModule } from '@angular/forms';
import { Subscription } from 'rxjs';
import { PROTOTYPE_REGISTRY } from '../../config/prototypes.registry';
import { PrototypeMeta } from '../../models/prototype-meta.model';
import { ProfileService } from '../../services/profile.service';
```

**Properties:**
- `allPrototypes: PrototypeMeta[]` -- full registry loaded once
- `filteredPrototypes: PrototypeMeta[]` -- after profile filter + search
- `searchQuery: string = ''` -- bound to search input with ngModel
- `currentProfile: string | null = null`
- `private subscription: Subscription`

**OnInit logic:**
1. Load `PROTOTYPE_REGISTRY` into `allPrototypes`
2. Subscribe to `ProfileService.currentProfile$`
3. On profile change: store profile, call `filterPrototypes()`

**filterPrototypes() method:**
1. Start with `allPrototypes`
2. Filter by profile: keep entries where `proto.profiles.includes(this.currentProfile)` (if profile is set)
3. Filter by search: if `searchQuery` is not empty, keep entries where `title.toLowerCase().includes(query)` OR `module.toLowerCase().includes(query)` (per D-18, search indexes title and module)
4. Sort by `dateAdded` descending (newest first, per HUB-03): `sort((a, b) => b.dateAdded.localeCompare(a.dateAdded))`
5. Assign to `filteredPrototypes`

**onSearchChange() method:**
Called on every keystroke (per D-17, instant filter, no debounce). Calls `filterPrototypes()`.

**clearSearch() method:**
Sets `searchQuery = ''` and calls `filterPrototypes()`.

**isNew(proto: PrototypeMeta): boolean method:**
Returns true if `dateAdded` is within the last 7 days (per HUB-04, D-06):
```typescript
const added = new Date(proto.dateAdded);
const now = new Date();
const diffDays = (now.getTime() - added.getTime()) / (1000 * 60 * 60 * 24);
return diffDays <= 7;
```

**getOwnerName(email: string): string method:**
Derives display name from email (per D-08):
```typescript
// "producto@dropi.co" -> "Producto"
// "juan.perez@dropi.co" -> "Juan Perez"
const local = email.split('@')[0];
return local.split('.').map(part =>
  part.charAt(0).toUpperCase() + part.slice(1)
).join(' ');
```

**getThumbnailPath(proto: PrototypeMeta): string method:**
Constructs the path to the thumbnail image. Since thumbnails live next to the component, and we're serving from assets, construct a relative path based on the prototype's location. For now, use a placeholder path pattern. The actual thumbnails will be generated by Plan 04's Playwright script.
```typescript
// Return path to thumbnail within the prototype's directory
// Since Angular serves from assets, construct the path:
return `assets/thumbnails/${proto.slug}.png`;
```
Note: Plan 04 will create thumbnails. For this plan, use a fallback placeholder approach:
- In the template, use `(error)="onThumbnailError($event)"` on the img tag
- The error handler sets `src` to a gradient placeholder div (hide img, show placeholder)

**OnDestroy:** Unsubscribe from profile subscription.

**Template (inline):**

```html
<div class="hub">
  <div class="hub__header">
    <div class="hub__title-row">
      <h2>Prototipos</h2>
      <span class="hub__count">{{ filteredPrototypes.length }} prototipos</span>
    </div>
    <div class="hub__search">
      <i class="pi pi-search"></i>
      <input
        type="text"
        [(ngModel)]="searchQuery"
        (ngModelChange)="onSearchChange()"
        placeholder="Buscar por titulo o modulo..."
        class="hub__search-input"
      />
    </div>
  </div>

  <!-- Empty state: no prototypes match search -->
  <div class="hub__empty" *ngIf="filteredPrototypes.length === 0 && searchQuery">
    <i class="pi pi-search"></i>
    <h3>No se encontraron prototipos para '{{ searchQuery }}'</h3>
    <button class="hub__clear-btn" (click)="clearSearch()">Limpiar busqueda</button>
  </div>

  <!-- Empty state: no prototypes for profile (no search active) -->
  <div class="hub__empty" *ngIf="filteredPrototypes.length === 0 && !searchQuery">
    <i class="pi pi-inbox"></i>
    <h3>No hay prototipos para este perfil</h3>
  </div>

  <!-- Card grid -->
  <div class="hub__grid" *ngIf="filteredPrototypes.length > 0">
    <a
      *ngFor="let proto of filteredPrototypes"
      [routerLink]="proto.route"
      class="hub__card"
      [title]="proto.description"
    >
      <!-- Thumbnail -->
      <div class="hub__card-thumbnail">
        <img
          [src]="getThumbnailPath(proto)"
          [alt]="proto.title"
          (error)="onThumbnailError($event, proto)"
          loading="lazy"
        />
        <div class="hub__card-placeholder" *ngIf="!proto._thumbnailLoaded">
          <i class="pi pi-image"></i>
        </div>
        <span class="hub__badge-nuevo" *ngIf="isNew(proto)">Nuevo</span>
      </div>

      <!-- Card body -->
      <div class="hub__card-body">
        <h3 class="hub__card-title">{{ proto.title }}</h3>
        <div class="hub__card-meta">
          <span class="hub__card-module">{{ proto.module }}</span>
          <span class="hub__card-separator">|</span>
          <span class="hub__card-owner">{{ getOwnerName(proto.owner) }}</span>
        </div>
        <div class="hub__card-date">{{ proto.dateAdded }}</div>
      </div>
    </a>
  </div>
</div>
```

Note on `proto._thumbnailLoaded`: Add a transient property to track image load state. Use `(load)="proto._thumbnailLoaded = true"` on the img tag. This is NOT part of the PrototypeMeta interface -- cast or use `any` for this UI-only property. The cleaner approach: use a `Set<string>` property `loadedThumbnails` in the component to track which slugs have loaded.
