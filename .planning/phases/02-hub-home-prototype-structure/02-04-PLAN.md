**Step 3: Create `scripts/generate-thumbnails.js`**

This is a plain Node.js script (NOT a Playwright test). It uses `@playwright/test` as a library.

```javascript
// scripts/generate-thumbnails.js
// Captures screenshots of each prototype route for Hub card thumbnails
// Usage: yarn generate-thumbnails
// Prerequisite: dev server running on localhost:4200

const { chromium } = require('playwright');  // Note: use 'playwright' package from @playwright/test
const fs = require('fs');
const path = require('path');

const REGISTRY_PATH = path.join(__dirname, '..', 'src', 'app', 'config', 'prototypes.registry.ts');
const OUTPUT_DIR = path.join(__dirname, '..', 'src', 'assets', 'thumbnails');
const BASE_URL = process.env.BASE_URL || 'http://localhost:4200';
const VIEWPORT = { width: 1280, height: 720 };
```

**Script logic:**

1. Parse `prototypes.registry.ts` to extract route and slug for each prototype. Use regex to extract the array entries (since this is a .ts file, parse with regex, NOT import):
   ```javascript
   const content = fs.readFileSync(REGISTRY_PATH, 'utf8');
   // Extract objects from the array using regex
   const entries = [];
   const regex = /slug:\s*'([^']+)'[\s\S]*?route:\s*'([^']+)'/g;
   let match;
   while ((match = regex.exec(content)) !== null) {
     entries.push({ slug: match[1], route: match[2] });
   }
   ```

2. Create output directory if it doesn't exist: `fs.mkdirSync(OUTPUT_DIR, { recursive: true })`

3. Launch Chromium browser (headless)

4. For each prototype entry:
   a. Check if `src/assets/thumbnails/{slug}.png` already exists AND was manually placed (per D-16: respect manual overrides). Simple heuristic: if the file exists and has a `.manual` marker file next to it, skip. Otherwise, overwrite.
   b. Create a new page, set viewport to 1280x720
   c. Navigate to `${BASE_URL}${entry.route}`
   d. Wait for network idle (page loaded)
   e. Handle auth: The dev server requires login. Two approaches:
      - **Option A (recommended):** Before capturing, set localStorage auth state to bypass login. Set `dropi_hub_user` with a mock user JSON, and set `dropi_hub_profile` in sessionStorage to `'admin'` (admin sees most content). Navigate to the route after setting storage.
      - **Option B:** Navigate to login first, but this requires Google OAuth which is not automatable.
      
      Use Option A: Before each page navigation, evaluate script to set auth state:
      ```javascript
      await page.goto(BASE_URL);
      await page.evaluate(() => {
        localStorage.setItem('dropi_hub_user', JSON.stringify({
          uid: 'thumbnail-bot',
          email: 'producto@dropi.co',
          displayName: 'Thumbnail Bot',
          photoURL: '',
          createdAt: new Date().toISOString()
        }));
        sessionStorage.setItem('dropi_hub_profile', 'admin');
      });
      await page.goto(`${BASE_URL}${entry.route}`);
      await page.waitForLoadState('networkidle');
      ```
   f. Take screenshot: `await page.screenshot({ path: outputPath, type: 'png' })`
   g. Log: `"Captured: {slug}.png"`

5. Close browser
6. Print summary: `"Generated {N} thumbnails in src/assets/thumbnails/"`
